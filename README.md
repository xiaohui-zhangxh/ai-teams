# AI Teams

用 AI CLI 工具组建你的 AI 团队——为每个角色预设专属提示词，一条命令即可切换身份。

支持多种 AI CLI 后端（Cursor Agent、Claude Code、Gemini CLI 等）和多语言提示词。

## 工作原理

每个角色由两部分组成：

- **角色提示词**（`roles/{locale}/` 目录）：按语言存放，并可继续按企业类型分层组织，定义角色的职责、工作风格和决策原则
- **启动入口**（`bin/{cli}-as`）：交互式选择角色，自动检测系统语言

启动时读取所选角色的提示词，传递给对应的 CLI 工具：

```bash
cursor-agent "$(cat roles/zh-CN/软件科技公司/01-产品策略负责人.md)"
```

## 命名规范

```
{cli}-as    # 交互式角色选择器（唯一入口，脚本名去掉 `-as` 即为 CLI 命令名）
```

| CLI 后端 | 入口 |
|----------|------|
| Cursor Agent | `cursor-agent-as` |
| Claude Code | `claude-code-as`（复制 `cursor-agent-as` 并重命名即可） |
| Gemini CLI | `gemini-as` |

## 安装

```bash
git clone <repo-url> ~/ai-teams
cd ~/ai-teams
bash install.sh
source ~/.zshrc   # 或 source ~/.bashrc
```

安装脚本会将 `bin/` 目录加入你的 `PATH`，之后在任意位置都可以直接运行 `cursor-agent-as`。

### 手动指定语言

通过环境变量 `AI_TEAM_LANG` 覆盖自动检测：

```bash
AI_TEAM_LANG=en cursor-agent-as    # 英文菜单与英文提示词
```

## 多语言机制

| 优先级 | 语言来源 | 示例 |
|--------|----------|------|
| 1（最高） | 环境变量 `AI_TEAM_LANG` | `AI_TEAM_LANG=en` |
| 2 | 系统语言 `$LANG` | `zh_CN.UTF-8` → `zh-CN` |
| 3（兜底） | 默认 | `en` |

角色文件按语言目录存放，各语言目录可继续按企业类型或经营变体分层（如 `roles/zh-CN/软件科技公司/`、`roles/zh-CN/自媒体运营企业/11-短视频型/`）。`cursor-agent-as` 会递归扫描当前语言目录下所有 `.md` 文件；首行 `# 标题` 作为菜单显示名，缺省则用文件名。若角色位于子目录中，菜单会显示为 `目录 / 角色名`。文件和子目录都建议使用两位数字前缀（如 `01-`、`02-`）控制显示顺序，菜单中会自动隐藏这些排序前缀。

## 多类型企业组织最佳实践

当前中文角色库已经内置两大企业类型：`软件科技公司` 与 `自媒体运营企业`。其中 `自媒体运营企业` 又细分为 `通用型`、`短视频型` 和 `知识付费型` 三套组织角色。各套目录都按**固定协作流**排序，强调能力分工而非行政头衔。

### 软件科技公司

按 **产品 -> 架构 -> 审查 -> 实现 -> 验收** 的固定协作流排序，角色强调能力分工而非行政层级：

| 顺序 | 角色 | 主要职责 | 主要对接 |
|------|------|----------|----------|
| 01 | 产品策略负责人 | 定义需求、MVP、验收标准与价值判断 | 架构师、验收 |
| 02 | 首席架构师 | 输出技术蓝图、选型与非功能约束 | 产品、审查、实现 |
| 03 | 技术审查官 | 挑战架构方案，控制复杂度与风险 | 架构师、实现 |
| 04 | 实现负责人 | 编码、补必要测试并交付可验证结果 | 架构、审查、验收 |
| 05 | 代码审计专家 | 深挖逻辑、安全、并发与性能问题 | 实现 |
| 06 | 技术验收专家 | 对照 PRD 做最终一致性验收 | 产品、实现 |

后续若扩展新的企业类型，直接新增类似目录即可，例如 `roles/zh-CN/自媒体运营企业/`，并沿用同样的两位数字前缀规则。

| 02 | 选题策划负责人 | 选题池、内容切口、脚本提纲与排期 | 策略、制作 |
| 03 | 内容制作负责人 | 成品制作、包装与多平台版本交付 | 策划、质检 |

| 05 | 增长运营负责人 | 渠道分发、互动、拉新转化与数据复盘 | 质检、商业化 |
| 06 | 商业化负责人 | 广告合作、产品变现与收入结构优化 | 策略、增长 |

## 添加新角色

1. 先选择语言和企业类型目录，例如 `roles/zh-CN/软件科技公司/`：

```bash
mkdir -p "roles/zh-CN/软件科技公司"
```

2. 用两位数字前缀创建新角色文件：

```bash
echo '# 交付经理' > "roles/zh-CN/软件科技公司/10-交付经理.md"
```

3. 下次运行 `cursor-agent-as` 时菜单会自动出现新角色，无需改 `bin/`。

> `en/` 下建议用 ASCII 文件名（如 `01-project-manager.md`），便于跨平台与脚本引用；中文目录下文件名可使用中文，只要扩展名为 `.md`。

> 如果同一企业类型下还要继续细分经营模式，也建议给子目录加两位数字前缀，例如 `11-短视频型`、`12-知识付费型`，便于控制菜单顺序。

## 添加新企业类型

1. 创建企业类型目录，例如：

```bash
mkdir -p "roles/zh-CN/连锁餐饮企业"
```

2. 按该企业的协作链条，用两位数字前缀创建角色文件：

```bash
echo '# 区域运营总监' > "roles/zh-CN/连锁餐饮企业/01-区域运营总监.md"
```

3. 下次运行 `cursor-agent-as` 时，新企业类型下的角色会自动出现在菜单中。

## 添加企业子类型

如果一个企业类型下还要继续沉淀不同经营模式，可在企业目录下继续创建带排序前缀的子目录，例如：

```bash
mkdir -p "roles/zh-CN/自媒体运营企业/13-播客型"
echo '# 节目主理人' > "roles/zh-CN/自媒体运营企业/13-播客型/01-节目主理人.md"
```

菜单中会显示为 `自媒体运营企业 / 播客型 / 节目主理人`。

## 扩展新 CLI 后端

以 Claude Code 为例，复制选择器即可（脚本从文件名推导 CLI 命令）：

```bash
cp bin/cursor-agent-as bin/claude-code-as
chmod +x bin/claude-code-as
```

角色提示词（`roles/`）所有 CLI 后端共享，无需重复维护。
